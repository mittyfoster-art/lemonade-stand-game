### Feature #32: Real-time Room Sync Bug Fix

**Problem Identified:**
- The `createGameRoom` function in `game-store.ts` was NOT calling `subscribeToRoom()` after creating a room
- This meant facilitators who created a room wouldn't receive real-time updates when players joined
- They would only see player updates if they left and rejoined the room

**Solution Implemented:**
- Added `get().subscribeToRoom()` call after room creation in `createGameRoom` function (line 1170)
- This ensures facilitators automatically subscribe to real-time Supabase updates
- Players joining from any device are now visible in real-time on the Facilitator Dashboard

**Code Change:**
```typescript
// In createGameRoom function, after syncing to backend:
// Subscribe to real-time updates for the newly created room
// so the facilitator can see players joining in real-time
get().subscribeToRoom()
```
